Take-Home Assessment – Testing Implementation

Module: SE3112 - Advanced Software Engineering (BSc. Hons in Computer Science - Y3S1)

Weight: 10% Group size: 4 students Time: 12 minutes No Selenium or Postman
Release Date: 31 March 2026
Due Date: 26 April 2026
VIVA Schedule: Last week of April (During Monday Lecture and Practical slots). Specific timeslots will be posted on Moodle.

Objective

Select an existing codebase—this can be a web application, a process optimization script, a parallel computing model, or a graphics/visualization project. Implement automated testing on this codebase using two distinct testing tools from two different categories. Deliver a group presentation with individual, feature-specific live code demos.

Tool & Category Selection

Select two categories from the list below, and choose one tool for each category. Do not use Selenium or Postman.

  • Unit/Component Testing: JUnit, PyTest, TestNG, Vitest, Mocha, RSpec
  • API/Integration Testing: RestAssured, Karate, Supertest
  • E2E / UI Testing: Cypress, Playwright, Robot Framework
  • Load/Performance Testing: JMeter, Gatling, K6, Locust
  • Behavior-Driven Development: Cucumber, Behave

Division of Work (Equal Scope)

To ensure equal contribution, each of the 4 students must take ownership of a distinct feature across the two chosen tools. For example:

  • Tool 1 (e.g., PyTest - Unit Testing):
    • Student 1: Implements and demos Parameterized Testing.
    • Student 2: Implements and demos Fixtures and Mocking.
  • Tool 2 (e.g., K6 - Load Testing):
    • Student 3: Implements and demos Spike/Stress Testing configurations.
    • Student 4: Implements and demos Data-driven payload testing (e.g., from CSV).

Deliverables & Presentation Format (12 Minutes)

  1. Group Intro (2 mins): Briefly explain the chosen codebase, the two testing categories selected, and why the chosen tools fit the project.
  2. Individual Demos (8 mins total - 2 mins per student): Each student shows their specific implemented feature in actual working code and runs it live.
  3. VIVA / Q&A (2 mins): Instructors will ask technical questions directed at individual students regarding their specific code and testing concepts.
  4. Submission package (Upload to Moodle before your slot):
    • Slides (max 5) or a 1-page architecture PDF
    • Source code (Project + Tests) with a README outlining who did which feature.
